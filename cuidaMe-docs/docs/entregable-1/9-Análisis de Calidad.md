## Introducción
El propósito fundamental de este documento radica en establecer un conjunto estructurado de directrices y medidas destinadas a garantizar que la aplicación de CuidaMe cumpla con los estándares claramente definidos.
## Acciones

|**REQUISITO**|**CRITERIO DE ACEPTACIÓN**|**MÉTODO DE VERIFICACIÓN**|**RESPONSABLE**|
| :-: | :-: | :-: | :-: |
|RI-01, RI-02, RF-06, RN-01,RN-02|El sistema de registro debe permitir a los usuarios crear cuentas con éxito, validar la información del usuario y almacenarla de manera segura en la base de datos.|Pruebas unitarias para verificar la funcionalidad del registro, pruebas de seguridad para proteger la información del usuario y pruebas de visualización  que verifiquen la funcionalidad.|||||
|RF-07|El usuario  debe ser capaz de visualizar y poder editar su perfil  con éxito.|Pruebas unitarias que verifiquen que se pueda obtener la información del usuario correctamente y que este pueda modificar sus datos y pruebas de visualización  que verifiquen la funcionalidad.|||||
|RI-03, RF-02, RN-03 y RN-04|El perfil de cuidador debe estar correctamente implementado de manera que este sea capaz de publicar sus propias ofertas de empleo.|Pruebas unitarias que verifiquen que el usuario cuidador pueda publicar ofertas correctamente y  pruebas de visualización  que verifiquen la funcionalidad.|||||
|RI-03 y RF-09|El usuario cuidador debe ser capaz de editar sus ofertas con éxito.|Pruebas unitarias que verifiquen que puedan modificar las ofertas correctamente y  pruebas de visualización  que verifiquen la funcionalidad.|||||
|RF-01|Se debe mostrar de forma efectiva el listado de ofertas activas y sus detalles y que se puedan filtrar según los criterios establecidos.|Pruebas unitarias para verificar que se puedan listar, detallar y filtrar las ofertas correctamente y  pruebas de visualización  que verifiquen la funcionalidad.|||||
|RF-04, RN-05|La solicitud de chat a través de una oferta debe estar clara y visible en todo momento en una oferta. |Pruebas unitarias que verifiquen que la solicitud de chat funcione correctamente y  pruebas de visualización  que verifiquen la funcionalidad.|||||
|RF-05|En el perfil del cuidador debe haber un espacio en el que aparezcan las solicitudes de chats pendientes y este tenga la capacidad de poder aceptar/denegar una solicitud con éxito.|Pruebas unitarias que verifiquen que las solicitudes de chat aparecen de forma correcta en el usuario cuidador y que la funcionalidad de aceptar/denegar funcione correctamente y  pruebas de visualización  que verifiquen la funcionalidad.|||||
|RF-09|El sistema debe constar con un chat en el que se puedan comunicar los cuidadores con las personas dependientes/sus familiares y que este funcione con éxito.|Pruebas unitarias que verifiquen que el chat funciona correctamente y  pruebas de visualización  que verifiquen la funcionalidad.|||||
|RF-10|El sistema debe contar con un espacio donde se puedan valorar a los cuidadores y esto sea visible para los clientes.|Pruebas unitarias que verifiquen que se puedan valorar correctamente a los cuidadores y que sea visible para todos y  pruebas de visualización  que verifiquen la funcionalidad.|||||
|RI-04|El sistema debe contar con un apartado en el que se encuentren los planes de suscripciones y estos mismos estén detallados de manera correcta con la información detallada ,clara y concisa de que incluye cada plan. |Verificar que existe un apartado en el que estén los planes de suscripción con la información necesaria para cada uno de forma detallada, clara y concisa y que se pueda acceder a dicho apartado y  pruebas de visualización  que verifiquen la funcionalidad.|||||
|RF-11|La pasarela de pago debe ser segura, cumplir con estándares de seguridad, Pruebas de integración con la pasarela de pago, pruebas de proporcionar una experiencia de pago sin problemas para los usuarios|Realizar pruebas de integración con la pasarela de pago y pruebas de seguridad para validar la seguridad de las transacciones y  pruebas de visualización  que verifiquen la funcionalidad.|||||
|RF-13|El sistema debe contar con un mapa en el que se puedan localizar a los cuidadores según una zona señalada con éxito.|Pruebas unitarias que verifiquen que la funcionalidad de localizar cuidadores en un mapa según una zona funcione de forma correcta y  pruebas de visualización  que verifiquen la funcionalidad.|||||
|RF-14|El sistema debe contar con una funcionalidad que sea poner en contacto entre cuidador y cliente por videollamadas con éxito.|Pruebas unitarias que verifiquen que se pueda realizar una videollamada correctamente y  pruebas de visualización  que verifiquen la funcionalidad.|||||
|RF-15|El usuario debe poder iniciar sesión con otras cuentas como Google, facebook..|Pruebas unitarias que verifiquen que el usuario puede loguearse en la aplicación de manera correcta y  pruebas de visualización  que verifiquen la funcionalidad.|||||
|RC-01|Comprobar cuántos usuarios es capaz de soportar la aplicación a la vez para ver si soporta la cantidad de usuarios calculados para que la aplicación sea rentable|Pruebas de carga para verificar si soporta el número de usuarios calculados para que la app sea rentable.|||||
|RC-02|<p>La aplicación web debe ser accesible, intuitiva y seguir estándares de [W3C.](https://www.w3.org/WAI/standards-guidelines/wcag/es)[](https://www.w3.org/WAI/standards-guidelines/wcag/es)</p><p>[](https://www.w3.org/WAI/standards-guidelines/wcag/es)</p>|Pruebas de accesibilidad que verifiquen que el estándar está correctamente aplicados|||||
